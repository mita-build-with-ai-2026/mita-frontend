# MITA - CONTEXTO DEL PROYECTO Y NEGOCIO

Fecha de actualizacion: 2026-05-30

## 1. Resumen ejecutivo

Mita es un agente inmobiliario con inteligencia artificial para ayudar a personas, emprendedores, pymes e inmobiliarias de Santa Cruz a encontrar el espacio correcto.

La version alineada para Build With AI 2026 es:

**Mita te ayuda a encontrar tu espacio con IA: para vivir, trabajar, vender, almacenar, operar o emprender.**

Para la hackathon, el foco principal es la mencion **Industria**, con una solucion aplicada a espacios productivos, comerciales y logisticos:

* locales comerciales,
* oficinas,
* depositos,
* galpones,
* talleres,
* espacios logisticos,
* espacios para emprendimientos,
* y propiedades residenciales como extension natural del producto.

Mita no debe presentarse como otro portal inmobiliario. Debe presentarse como una capa inteligente encima de un mercado local fragmentado: entiende publicaciones desordenadas, convierte texto informal en datos estructurados, permite buscar en lenguaje natural, compara opciones y explica por que cada espacio conviene o no.

## 3. Problema

En Santa Cruz, encontrar un espacio adecuado sigue siendo un proceso lento, manual y desordenado.

Las ofertas inmobiliarias estan dispersas entre:

* portales inmobiliarios,
* Facebook Marketplace,
* grupos de Facebook,
* grupos de WhatsApp,
* publicaciones de inmobiliarias,
* contactos personales,
* anuncios repetidos,
* publicaciones antiguas,
* imagenes con poca informacion,
* textos incompletos.

Para una persona que busca vivienda, esto ya es frustrante. Para un emprendimiento o pyme, es todavia mas critico porque no solo importan precio y zona. Tambien importan criterios operativos:

* metros cuadrados,
* acceso para vehiculos o camiones,
* visibilidad comercial,
* flujo de personas,
* cercania a avenidas,
* servicios basicos,
* seguridad,
* parqueo,
* tipo de actividad permitida,
* condiciones de alquiler,
* expensas,
* energia disponible,
* compatibilidad con el uso previsto.

Hoy esa informacion suele estar incompleta, escrita de forma distinta o directamente ausente.

Frase clave del problema:

> "El mercado tiene ofertas, pero no tiene informacion suficientemente estructurada para decidir rapido."

La gente no busca espacios solo con filtros. Busca con contexto.

Un filtro tradicional pregunta:

* zona,
* precio,
* tipo de propiedad,
* dormitorios,
* metros cuadrados.

Pero una necesidad real suena asi:

* "Necesito un local comercial cerca de una avenida transitada, maximo 800 dolares, ideal para una cafeteria."
* "Busco un deposito o galpon de al menos 200 m2 en zona norte, con acceso para camion y seguridad."
* "Necesito una oficina pequena para una startup de 5 personas cerca de Equipetrol."
* "Quiero un taller donde pueda operar sin molestar a vecinos y con buen acceso."
* "Busco un departamento cerca de mi trabajo, con garaje y que acepte mascota."

Mita debe entender ese contexto y convertirlo en criterios accionables.

## 5. Solucion

Mita es un agente inmobiliario con IA que convierte una busqueda desordenada en una decision clara.

Flujo principal:

1. Mita importa ofertas desde una fuente demo o externa.
2. La IA normaliza publicaciones desordenadas.
3. Mita extrae datos importantes: precio, zona, tamano, tipo de espacio, condiciones, servicios y caracteristicas operativas.
4. El usuario busca en lenguaje natural.
5. Mita interpreta la intencion y transforma la busqueda en criterios estructurados.
6. Mita recupera opciones relevantes usando filtros, scoring y busqueda semantica.
7. Mita ordena resultados por compatibilidad.
8. Mita explica por que cada opcion encaja o no.
9. Mita permite comparar alternativas.
10. El usuario contacta por WhatsApp.

Mita no solo lista propiedades. Ayuda a decidir.

## 8. Uso de inteligencia artificial

Mita debe mostrar IA real, no solo un buscador con texto.

Capacidades principales de IA:

### 8.1 Normalizacion de publicaciones

Entrada desordenada:

> "alquilo local zona norte 800$ ideal negocio avenida transitada bano privado consultas whatsapp"

Salida estructurada:

```json
{
  "tipoPropiedad": "LOCAL_COMERCIAL",
  "tipoUsoEspacio": "COMERCIAL",
  "zona": "zona norte",
  "precio": 800,
  "moneda": "USD",
  "aptoPara": ["cafeteria", "tienda", "showroom"],
  "caracteristicasOperativas": {
    "sobreAvenida": true,
    "altoTransito": true,
    "banos": 1
  },
  "requisitosFaltantes": [
    "metros cuadrados",
    "condiciones de alquiler",
    "parqueo",
    "servicios basicos",
    "permiso de actividad comercial"
  ]
}
```

### 8.2 Busqueda conversacional

El usuario escribe:

> "Necesito un local comercial cerca de una avenida transitada, maximo 800 dolares, ideal para abrir una cafeteria."

Mita extrae:

* tipo de espacio: local comercial,
* uso esperado: cafeteria,
* presupuesto maximo: 800 USD,
* prioridad: visibilidad y transito,
* zona deseada: avenida o zona comercial,
* advertencias necesarias: permisos, servicios, parqueo, condiciones.

### 8.3 Ranking inteligente

Cada propiedad recibe un porcentaje de match segun compatibilidad con la necesidad del usuario.

Ejemplo:

* 94% match: local en avenida transitada, dentro del presupuesto, compatible con cafeteria.
* 86% match: local con buena zona, pero no confirma parqueo.
* 72% match: oficina adaptable, pero no ideal para atencion al publico.

### 8.4 Explicaciones y alertas

Mita explica:

> "Esta opcion encaja porque esta en una zona de alto transito, esta dentro del presupuesto y el tipo de espacio es compatible con atencion al publico. Como alerta, la publicacion no menciona si cuenta con permiso para actividad gastronomica."

### 8.5 Comparador

El usuario puede preguntar:

> "Cual me conviene mas si quiero abrir una cafeteria pequena y necesito buena visibilidad?"

Mita responde con una recomendacion y tradeoffs claros.

## 10. Arquitectura

Arquitectura general:

```txt
React Frontend
    |
Node.js / Express API
    |
PostgreSQL
    |
Servicios de IA
```

Componentes:

* **Frontend web:** busqueda, resultados, detalle de espacio, comparador, login admin y flujo demo.
* **Backend API:** endpoints, autenticacion, propiedades, leads, importaciones, normalizacion y ranking.
* **IA:** extraccion de criterios, normalizacion, explicaciones, comparacion y busqueda semantica.
* **Base de datos:** propiedades, fuentes, importaciones, busquedas, resultados, embeddings y leads.

Stack recomendado para el MVP:

* React,
* Vite,
* Node.js / Express,
* PostgreSQL,
* Gemini API,
* WhatsApp link para contacto,
* dataset demo curado.
