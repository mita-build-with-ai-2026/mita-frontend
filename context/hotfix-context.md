# HOTFIX DE ENFOQUE — MITA

## 1. Motivo del hotfix

Después de revisar los lineamientos oficiales del Build With AI 2026, identificamos que el proyecto debe enmarcarse dentro de una de las menciones oficiales del hackathon: Agua, Energía, Industria o Agro.

La idea original de Mita estaba planteada como un agente inmobiliario con IA para encontrar alquileres en Santa Cruz, principalmente departamentos y vivienda. Aunque el problema es real y local, ese enfoque puede parecer demasiado residencial y podría no encajar de forma directa con las menciones oficiales.

Por eso hacemos un hotfix de enfoque:

Mita ya no será presentada únicamente como un buscador de departamentos.

Mita será presentada como:

**Un agente inmobiliario con IA para encontrar espacios en Santa Cruz: espacios para vivir, trabajar, vender, almacenar, operar o emprender.**

Para la hackathon, el foco principal será:

**Mita para espacios productivos, comerciales y logísticos en Santa Cruz.**

Esto nos permite entrar en la mención **Industria**, porque atacamos un problema relacionado con digitalización, logística, operación comercial, búsqueda de infraestructura física y eficiencia para emprendedores, pymes, comercios, talleres y operadores locales.

## 3. Nueva definición del proyecto

Mita es un agente inmobiliario con IA que ayuda a personas, emprendedores y empresas de Santa Cruz a encontrar el espacio correcto según sus necesidades.

Mita recopila ofertas inmobiliarias dispersas, las entiende con inteligencia artificial, las convierte en datos estructurados y recomienda las mejores opciones según presupuesto, zona, tamaño, características, servicios, accesibilidad y tipo de uso.

Para el MVP del hackathon, Mita se enfocará especialmente en espacios comerciales y productivos:

* oficinas,
* locales comerciales,
* depósitos,
* galpones,
* talleres,
* espacios logísticos,
* espacios para emprendimientos,
* y también propiedades residenciales como extensión natural del producto.

## 4. Problema corregido

En Santa Cruz, encontrar un espacio adecuado para operar, vender, almacenar, trabajar o emprender sigue siendo un proceso lento, manual y desordenado.

Las ofertas inmobiliarias están dispersas entre portales, Facebook, WhatsApp, grupos, inmobiliarias y publicaciones informales. Además, muchas publicaciones están incompletas o mal estructuradas.

Para una persona que busca vivienda esto ya es frustrante, pero para un emprendimiento o pequeña empresa puede ser aún más crítico, porque no solo importa el precio o la zona. También importan factores como:

* metros cuadrados,
* acceso para vehículos o camiones,
* ubicación estratégica,
* seguridad,
* servicios básicos,
* tipo de actividad permitida,
* cercanía a avenidas,
* condiciones de alquiler,
* depósitos,
* oficinas,
* local comercial,
* taller,
* galpón,
* parqueo,
* visibilidad comercial,
* conectividad logística.

Hoy esa información no está bien organizada.

El usuario debe revisar muchas publicaciones manualmente, preguntar por WhatsApp datos que deberían estar claros y comparar opciones sin una herramienta que entienda realmente lo que necesita.

---

## 5. Solución corregida

Mita resuelve este problema con un agente inmobiliario basado en IA.

El flujo es:

1. Mita importa ofertas inmobiliarias desde fuentes externas o demo.
2. La IA normaliza publicaciones desordenadas.
3. Mita extrae datos importantes como precio, zona, tamaño, tipo de espacio, condiciones y características.
4. El usuario busca en lenguaje natural.
5. Mita entiende la intención de búsqueda.
6. Mita recupera opciones relevantes usando búsqueda semántica, filtros y ranking.
7. Mita explica por qué cada opción encaja o no.
8. El usuario compara alternativas.
9. El usuario contacta por WhatsApp.

Ejemplo de búsqueda:

“Necesito un local comercial cerca de una avenida transitada, máximo 800 dólares, ideal para abrir una cafetería.”

Otro ejemplo:

“Busco un depósito o galpón de al menos 200 m² en zona norte, con acceso para camión y seguridad.”

Otro ejemplo:

“Necesito una oficina pequeña para una startup de 5 personas cerca de Equipetrol o la UCB.”

Mita no solo lista propiedades.

Mita entiende necesidades de espacio.

## 10. Demo corregida

La demo debe mostrar este flujo:

### Paso 1: Importar ofertas

Mostrar que Mita puede importar ofertas desde una fuente demo o externa.

Mensaje:

“Primero Mita recopila ofertas dispersas y las convierte en datos útiles.”

### Paso 2: Normalizar con IA

Mostrar una publicación desordenada:

“alquilo local zona norte 800$ ideal negocio avenida transitada baño privado consultas whatsapp”

Mita la convierte en:

* tipo: local comercial,
* zona: zona norte,
* precio: 800 USD,
* uso sugerido: negocio/comercio,
* características: avenida transitada, baño privado,
* información faltante: metros cuadrados, condiciones de alquiler, parqueo, servicios.

### Paso 3: Buscar en lenguaje natural

Usuario escribe:

“Busco un local comercial cerca de una avenida transitada, máximo 800 dólares, ideal para abrir una cafetería.”

Mita detecta:

* tipo de espacio: local comercial,
* presupuesto máximo: 800 USD,
* uso: cafetería,
* prioridad: visibilidad, ubicación, tránsito de personas,
* zona deseada: avenida o zona comercial.

### Paso 4: Resultados con match

Mita muestra resultados:

* 94% match: local en avenida transitada, dentro del presupuesto.
* 86% match: local con buena zona, pero no confirma parqueo.
* 72% match: oficina adaptable, pero no ideal para cafetería.

### Paso 5: Explicación

Mita explica:

“Esta opción encaja porque está en una zona de alto tránsito, está dentro del presupuesto y el tipo de espacio es compatible con atención al público. Como alerta, la publicación no menciona si cuenta con permiso para actividad gastronómica.”

### Paso 6: Comparador

Usuario pregunta:

“¿Cuál me conviene más si quiero abrir una cafetería pequeña y necesito buena visibilidad?”

Mita recomienda una opción y explica tradeoffs.

### Paso 7: Contacto

Botón:

“Contactar por WhatsApp.”

## 12. Hotfix técnico necesario

El cambio de enfoque no requiere rehacer toda la arquitectura.

La arquitectura se mantiene:

* React JS,
* Node.js / Express,
* PostgreSQL,
* Gemini API,
* scraping/importación,
* normalización con IA,
* RAG-lite,
* scoring,
* comparador,
* WhatsApp leads.

Lo que sí debemos ajustar es el modelo de datos para que Propiedad no sea solo residencial.

### Cambio principal en base de datos

La tabla Propiedad debe soportar varios tipos de espacio:

Antes:

* departamento,
* garzonier,
* casa,
* habitación,
* monoambiente.

Ahora:

* departamento,
* casa,
* oficina,
* local comercial,
* depósito,
* galpón,
* taller,
* espacio logístico,
* terreno,
* otro.

### Campos adicionales recomendados

Para no complicar demasiado la base de datos, agregamos pocos campos flexibles:

* tipoUsoEspacio
* aptoPara
* caracteristicasOperativas
* requisitosFaltantes

Estos campos pueden ser JSONB o texto estructurado.

No necesitamos crear muchas tablas nuevas.
